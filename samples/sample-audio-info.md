# サンプル音声ファイル情報

## 概要

実際の音声ファイルはプライバシー保護のため含まれていませんが、以下のような音声ファイルを使用することを想定しています。

## 推奨フォーマット

### ファイル形式
- WAV (推奨)
- MP3
- M4A

### 音質設定
- **サンプリングレート**: 16kHz 以上
- **ビット深度**: 16-bit
- **チャンネル**: モノラル または ステレオ
- **ビットレート**: 128kbps 以上（MP3の場合）

### ファイルサイズ
- **最大**: 100MB
- **推奨**: 10-50MB
- **時間**: 30秒〜5分

## サンプル音声の作成方法

### オプション1: テキスト読み上げ (TTS)

Azure AI Speech の TTS を使用して音声を生成:

```bash
# Azure CLI でサンプル音声を生成（例）
az cognitiveservices speech synthesize \
  --voice "ja-JP-NanamiNeural" \
  --text "いらっしゃいませ。本日はどのようなお車をお探しですか？" \
  --output demo-sales-1.wav
```

### オプション2: 録音

実際の商談をシミュレートして録音:
- 2名で役割分担（営業担当と顧客）
- `demo-session-001.json` または `demo-session-002.json` のスクリプトを読む
- スマートフォンやPCのマイクで録音

### オプション3: ダミー音声ファイル

テスト用に無音または簡単な音声ファイルを生成:

```bash
# ffmpeg で30秒の無音ファイルを生成
ffmpeg -f lavfi -i anullsrc=r=16000:cl=mono -t 30 -q:a 9 -acodec pcm_s16le demo-silence.wav
```

## サンプル音声の配置

作成した音声ファイルは以下に配置:

```
samples/
├── audio/
│   ├── demo-session-001.wav
│   ├── demo-session-002.wav
│   └── README.md
```

## デモモードでの使用

デモモードでは実際の音声ファイルは不要です:
- アップロードUIでファイルを選択しても、実際にはアップロードされません
- モックサービスが事前定義された文字起こし結果を返します
- 音声処理をスキップして即座に結果が表示されます

## 本番モードでの使用

本番モード（Azure接続あり）でテストする場合:
1. 上記の方法で音声ファイルを作成
2. アップロードUIから音声ファイルを選択
3. Azure Speech Service で実際の文字起こしが実行されます

## プライバシー注意事項

⚠️ **重要**: 
- 実際の顧客情報を含む音声は使用しないでください
- デモ用には架空の名前とシナリオを使用してください
- 本番環境では必ず顧客の同意を取得してください
- 個人情報保護法（GDPR、個人情報保護法等）を遵守してください
