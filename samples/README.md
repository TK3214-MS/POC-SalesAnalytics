# サンプルデータ

このディレクトリには、デモモード用のサンプルデータが含まれています。

## サンプルセッション

### demo-session-001.json
- **顧客**: 山田太郎
- **ステータス**: 成約 (won)
- **概要**: 5人家族向けSUVの商談。予算400万円で7人乗りSUVを提案し、試乗予約まで進めた成功事例。
- **ポイント**:
  - 家族構成のヒアリング
  - 予算に合わせたローンプラン提示
  - 具体的な次のアクション（試乗予約）

### demo-session-002.json
- **顧客**: 佐藤花子
- **ステータス**: 保留中 (pending)
- **概要**: 通勤用コンパクトカーの商談。予算オーバーで即決には至らず、検討中。
- **ポイント**:
  - 予算確認のタイミング課題
  - フォローアップの重要性
  - 代替案提示の必要性

## データ構造

各JSONファイルは以下の情報を含みます：

- **基本情報**: セッションID、ユーザーID、店舗ID、顧客名、作成日時
- **文字起こし**: 話者分離された会話内容（タイムスタンプ付き）
- **PII マスキング**: 個人情報をマスキングしたテキスト
- **感情分析**: 全体および各発言の感情スコア
- **要約**:
  - 重要ポイント
  - 懸念事項
  - 次のアクション
  - 成功要因
  - 改善点
  - 引用
- **アウトカム**: won（成約）/ lost（失注）/ pending（保留）/ canceled（キャンセル）
- **SharePoint URL**: アップロードされた文書のURL

## 使用方法

### バックエンド（モック）
`api/FunctionsApp/AI/Mock/` および `api/FunctionsApp/Data/Mock/` のモック実装がこれらのデータ構造を返します。

### フロントエンド（モック）
`frontend/src/app/api-proxy.ts` の `getMockResponse` 関数がこれらのデータを返します。

### 音声ファイル（参考）
実際の音声ファイルは含まれていませんが、以下のような形式を想定：
- **フォーマット**: WAV, MP3, M4A
- **長さ**: 30秒〜5分程度
- **内容**: 自動車ディーラーでの商談会話（日本語）
- **話者**: 2名（営業担当と顧客）

## カスタマイズ

新しいサンプルセッションを追加する場合：
1. 既存のJSONファイルをコピー
2. ID、顧客名、日時を変更
3. 文字起こし内容を更新
4. 要約と感情分析を調整
5. `api/FunctionsApp/Data/Mock/MockCosmosRepository.cs` にデータを追加（オプション）
